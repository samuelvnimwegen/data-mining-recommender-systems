# data/processed — Processed datasets

Primary function

- Stores cleaned, normalized, and encoded datasets produced by the dataset cleaner (for example: `movies_cleaned.csv`, `ratings_train_cleaned.csv`).

Architectural purpose

- Cache preprocessed outputs to avoid re-running expensive transformations during iterative development.
- Provide files that are safe to load directly into model training pipelines.

Guidelines

- Files in this directory are generated by `src.dataloader.DatasetCleaner` (or the `main.py` CLI) and should not be edited manually.
- If you want to refresh the processed data, re-run the cleaning pipeline and commit only intentional, reproducible outputs.

Quick checks

```bash
# Show processed files
ls -la data/processed
# Preview a processed movies file
head -n 5 data/processed/movies_cleaned.csv
```

Notes

- Keep processed data relatively small in the repository; if files grow large, prefer using an external storage or artifact store and reference their paths in `src/configs/config.py`.
